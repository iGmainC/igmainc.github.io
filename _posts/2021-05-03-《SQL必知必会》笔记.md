# 了解 SQL

本章节主要介绍 SQL 究竟是什么，它能做什么事情。 

## 数据库基础

### 数据库

数据库是一个以某种有组织的方式存储的数据集合。是一种数据存储、查询模式的抽象，从使用者的角度来说你不需要关注它内部具体是怎么实现的。

从本质上来说*数据库*这个名称指的是存储数据的容器，但这和我们经常听到的“MySql 数据库……”的印象不太一样，人们通常用数据库这个术语来代表他们使用的数据库软件，这是不正确的。MySql 应该被成为数据库管理系统（Data Base Manage System），简称为 DBMS。

数据库是通过 DBMS 创建和操纵的容器，而具体它究竟是什么，形式如何，这你不需要关心（至少菜鸡不需要关心）。

### 表（Table）

表是一种结构化的文件，可用来存储某种特定类型的数据（简单说就是 Excel 加了给表头）。

存储在表中的数据应是同一种类型的数据，**绝不应将不相关的数据存放到同一个数据库表中**。

数据库中的每个表都有一个名字来标识自己。这个名字是唯一的，即同一个数据库中没有其他表具有相同的名字。

表具有一些特性，这些特性定义了数据在表中如何存储，包含存储什么样的数据，数据如何分解，各部分信息如何命名等信息。描述表的这组信息就是所谓的模式（schema）

> 迷惑
> 
> 这个模式应该指的就是表结构，但是后文中“模式可以用来描述数据库中特定的表，也可以用来描述整个数据库（和其中表的关系）”的“描述整个数据库”有点迷惑

### 列和数据类型

表由列组成。列存储表中某部分的信息。
列就是表中的一个字段。所有表都是由一个或多个列组成的。

每个列都有相应的数据类型。数据类型（datatype）定义了列可以存储哪些数据种类。数据类型限定了可存储在列中的数据种类（例如，防止在数值字段中录
入字符值）。数据类型还帮助正确地分类数据，并在优化磁盘使用方面起
重要的作用。因此，在创建表时必须特别关注所用的数据类型。

### 行

一行就是一条符合该表表结构的数据。

### 主键（primary key）

一个列（或一组列），其值能够**唯一标识**表中每一行数据。唯一标识表中每行的这个列（或这几列）称为主键。

虽然并不总是需要主键，应该总是定义主键。

定义主键的要求：

- 值不重复，且值是唯一的
- 不为空
- 值不允许修改或更新
- 主键值不能重用（如果某行从表中删除，它的主键不能赋给以后的新行）。

> 也可以一起使用多个列作为主键。在使用多列作为主键时，上述条件必须应用到所有列，所有列值的组合必须是唯一的（但单个列的值可以不唯一）。

## 什么是 SQL

SQL 是 Structured Query Language（结构化查询语言）的缩写。SQL 是一种专门用来与数据库沟通的语言。

SQL 有如下的优点:

- 基本通用，基本上各个 DBMS 都支持 SQL，但不同的 DBMS 有细微的差别（亿点点）
- 简单易学，典型的易于上手难以精通
- 强大，难以精通的原因
